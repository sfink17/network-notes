Network Application Architecture

There are two main structures used to allow applications running on different hosts to communicate over a network. The two architectures are client-server and peer to peer, or P2P.

Client-Server Model

In the client server model, there is a central machine called the server which "serves" things to other machines in the network. These items are typically documents; web pages.

This model allows users to send and recieve data across the network simply by downloading and uploading items to the central sever(s).

Peer to Peer Model

In the peer to peer model, each user in the network must establish an individual connection to each other user. This type of connection is useful when real-time data streaming is important, such as in a multiplayer game.

Pros and Cons of Peer to Peer
Pros Cons
No critical central connection Loss of anonymity
Direct connection to other end users Requires address of every user in the network
Harder to secure more individual connections

In my own experience, P2P connections are mostly used now for playing online games. This type of connection allows for less delay because there are no intermediary links for the data to traverse.
