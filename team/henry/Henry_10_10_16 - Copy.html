Multiplexing and Demultiplexing

  1. A multiplexing/demultiplexing service is needed for all computer networks.
  2. The transport layer has the job of delivering the data from the network layer to all the appropriate application processes.
  3. In order to send data to the appropriate application out of those running on a single machine, a number of steps must occur.
  4. The transport layer has a number of attributes.
    • Each applicaton has one or more dedicated sockets, therefore the transport layer must merely know the appropriate socket for delivery.
    • Each socket is either a UDP or TCP socket.
    • Each transport layer segment has a number of fields which designate the appropriate socket.
    • The job of determining the appropriate socket and parsing off the correct segments falls to is called demultiplexing.
    • The process of examining transport layer segments and concatenating them to pass to the network layer is called multiplexing.
  5. Each port number on a computer is a 16-bit number ranging from 0 to 65535.
  6. Source and destination port number fields are sockets on a machine which are designated for either transmission or reception, respectively.
  7. The port numbers ranging from 0 to 1023 are called well-known port numbers.
  8. To summarize:
    • Each socket is designated a port number.
    • When a segment arrives, the transport layer examines the destination port number and and send the segment to the correct socket.
    • The data in the segment passes through the port and to the application.

Connectionless Multiplexing and Demultiplexing

There are a few important distinctions to make in this section as compared to the previous one.

  1. When a UDP socket is created, it is automatically unassigned a non-well-known port number.
  2. A UDP designated segment includes a "return to sender" identifier of the source's port number.
