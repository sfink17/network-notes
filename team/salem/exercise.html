Q&A

  • Give a brief definition of a protocol?

    • Answer: The format and the order of messages exchanged between two or more communicating entities, as well as the actions taken on the transmission and/or receipt of a message or other event.

  • Who were the two people who built on Adramson's multiple-access protocal work when they developed the Ethernet protocol work when they developed the Ethernet protocol for wire-based shared broadcast networks?

  • Robert Metcalfe and David Boggs
  • Norman Abramson and Minitel
  • Tim Berners-Lee and Ted Nelson
  • Donald Davies and Roger Scantlebury
    • Answer: Robert Metcalfe and David Boggs

  • What is the difference between a virus and a worm

    • Answer: Require some form of user interaction to infect the user's device, but a worm can enter a device without any explicit user interaction.

  • Match the Layers with the correct packet names.

    • Answer:
      • Application : message
      • Transport : segment
      • Network : datagrams
      • Link : frames
      • Physical : none

  • What are cookies? Where are they stored?

    • Answer: They allow sites to keep track of users. They are stored in the user's end system.

  • What are the major difference between time-division multiplexing(TDM) and frequency-division multiplexing(FDM)?

    • Answer: FDMs are frequency spectrum of a link that is divided up among the connections established across the link, whereas; TDMs are based on time that are divided into frames of fixed duration which are divided into time slots.

  • What are the two fundamental approaches to moving data through a network of links and switches? Why would you use one over the other?

    • Answer: Circuit switching and packet switching. Packet over circuit because dedicated circuits are idle while nothing is going through.

  • What are the criterias for whether you want to use a TCP or UDP?

    • Reliable data transfer, throughput, timing, and security.

Q&A

  1. What are the two requirements for transport-layer multiplexing? (page 192)
    1. Sockets have unique identifiers.
    2. Each segment has special fields that indicate the socket to which the segment is to be delivered.
  2. Give a quick definition for demultiplexing and multiplexing. (page 191)
    • Demultiplexing: The receiving end, the transport layer examines these fields to identify the receiving socket and then directs the segment to that socket.
    • Multiplexing: Gathering data chunks at the source host from different sockets, encapsulating each data chunk with header information
  3. Why would you chose UDP over TCP? (page 199)
    • as soon as an application process passes data to UDP, UDP will package the data inside a UDP segment and immediately pass the segment to the network layer.
    • UDP just blasts away without any formal preliminaries.
    • TCP segment has 20 bytes of header overhead in every segment, but UDP only has 8 bytes of overhead.
  4. What is checksums and how is it used? (page 202 to page 204)
    • It is 16-bits that is added to a packet and is used to prevent corruption.
    • By converting all the 0s to 1s and converting all 1s to 0s. When added it should produce 1111111111111111. If there is a 0 in the sum, then there is an error in that packet.
  5. Match the correct rdt with the correct added feature or basic feature. (page 206 to page 215)
    • Answer:
      • rdt 1.0
        • sender and receiver.
        • Just one state.
        • Perfectly reliable.
      • rdt 2.0
        • Positive acknowledgments and negative acknowledgments.
        • Reliable data transfer protocols based on such retransmission are known as ARQ (Automatic Repeat reQuest) protocols.
          • Error detection
          • Receiver feedback
          • Retransmission
        • This deals with the problem of corrupt packets.
      • rdt 2.1
        • Field that has a sequence number into the field, which is 1-bit sequence number. It will allow the receiver to know whether the sender is resending the previously transmitted packet. This is to deal with the problem of ACK or NAK packets being corrupt.
      • rdt 2.2
        • The receiver must now include the sequence number of the packet being acknowledged by an ACK message
        • The sender must now check the sequence number of the packet being acknowledged by a received ACK message.
      • rdt 3.0
      • Countdown Timer, which needs to be able to:
        • Start the timer each time a packet is sent.
        • Respond to a timer interruption.
        • Stop the timer.
      • This is to deal with lost packets.
  6. What are the names of the three major components of a TCP congestion-control algorithm? (page 272 to page 277)
    • Answer:
    • Slow Start
    • Congestion Avoidence
    • Fast Recovery
  7. What is the well-known port numbers. And, what are they used for? (page 192)
    • 0 to 1023.
    • Those ports are used for things such as HTTP (port number 80) and FTP (port number 21).
  8. How does sequential number help with rdt? (page 210)
    • It helps distinguish between packets and to keep track of which ACK or NAK responded to which packet.
  9. What is a three-way handshake? (page 232)
    1. The client first sends a special TCP segment.
    2. The server responds with a second special TCP segment
    3. The client responds again with a third special segment, confirming their connection.
  10. What is a congestion window? (page 269)
    • Imposes a constraint on the rate at which a TCP sender can send traffic into the network, denoted cwnd.

Two Problems

  1. P29 pg 294:

      • Question: Why is it necessary for the server to use a special initial sequence number in the SYNACK?
      • Answer: To make sure that the user trying to connect is an actual person and not part of an attack because the hash that is stored in a cookie is gained from the source and destination IP and port numbers of the SYN segment.
      • Question: Suppose an attacker knows that a target host uses SYN cookies. Can the attacker create half-open or fully open connections by simply sending an ACK packet to the target? Why or why not?
      • Answer: No, you cannot do a half-open connection, however; if you sniffed the network and got the packet and etc. then you can spoof and make a full connection with the ACK.
      • Question: Suppose an attacker collects a large amount of initial sequence numbers sent by the server. Can the attacker cause the server to create many fully open connections by sending ACKs with those initial sequence numbers? Why?
      • Answer: Yes, because you can just spoof yourself and use the initial sequential numbers to make full-connections.

As the stop-and-wait protocol has very low channel utilization (shown in the crosscountry example), the designers of this application let the receiver keep sending back a number (more than two) of alternating ACK 0 and ACK 1 even if the corresponding data have not arrived at the receiver. Sample Example

  • We will use the graph you see below to make a least cost path and a forwarding table for A.
  • For clarification:
    • D(v): cost of the least-cost path from the source node to destination v as of this iteration of the algorithm.
    • p(v): previous node (neighbor of v) along the current least-cost path from the source to v.
    • N': subset of nodes; v is in N' if the least-cost path from the source to v is definitively known.
2,a in the graph above means that 2 is the distance from the root (A), and a is the previous node.
All of the distance from A starts at infinity. We start from A and grab all of A's neighbors(B, C, and D).
  • The distance between A and B is 2 which is less than infinity, so B's distance is switched to 2 and the previous node is A.
  • The distance between A and C is 3 which is less than infinity, so C's distance is stwitched to 3 and the previous node is A.
  • The distance between A and D is 4 which is less than infinity, so D's distance is stwitched to 4 and the previous node is A.
  • E and F are not direct neighbors of A, so their distance are infinity
A is added to N' which indicates that A has been checked.
B is the closest to the starting node A and that is not already in N'. So it will be checked next.
  • C has a distance of 7 from B and the distance from A to C is 3, so C's status is not updated because 7 is greater than 3.
  • F has a distance of 7 from B and the distance is currently infinity, so F's status is updated to 7 with the previous node set to B, because 7 is less than infinity.
  • A is in N' , which means that A has already checked all of its neighbors.
C is the closest to the starting node A and is not in N'. So C will be checked next.
  • D has a distance of 8 from C and the distance from A to D is 4, so D's status is not updated because 8 is greater than 4.
  • E has a distance of 4 from C and the distance is currently infinity, so E's status is updated to 4 with the previous node set to C, because 4 is less than infinity.
  • F has a distance of 6 from C and the distance from B to F is 7, so F's status is updated to 6 with the previous node set to C, because 6 is less than 7.
D is the closest to the starting node A and is not in N' (Note: we could have also chosen E). So C will be checked next.
  • E has a distance of 8 from D and the distance from A to D is 4, so D's status is not updated because 8 is greater than 4.
  • Again, C and A are not included because they are already in N'.
E is the closest to the starting node A and is not in N'. So E will be checked next.
  • F has a distance of 5 from E and the distance from C to F is 6, so F's status is updated to 5 with the previous node set to E, because 5 is less than 6.
Finally, F is the only node that is not in N'. So F will be checked.
  • Since all of the F's neighbors are in N', we do not do anything with F.
  • Now the least cost path has been found and we can make a forwarding table.
  • On the left side we have the destinations and on the left side we have the link we are going to use to send the packet (Note: each node has its own forwarding table, so this node only needs to know which node to send the packet to that is on the shortest path to the destination.
