Lecture

  • Finite State Machine:
    1. An abstract machine that can be in one of finite number of states.
    2. A set of states.
    3. Alphabet — A set of symbols.
    4. Transition Function: A set of rules (Can be given in a table) if in state X and looking at symbol alpha, then next stat is y. (Given a particular state and a particular symbol/imput, then transition to another particular state)
    5. Starting State: Need to know starting state to know where to go next.
    6. Set of accepting states: know whether a set of states is allowed or not (good/bad).
    7. Can be used in the development of a compiler.
    8. Seperate FSM for receiver and sender
  • Regular Expressions: Way of decribing regular languages.
  • Regular Languages: What finite state machines recognize.
  • 4 Types of Languages
  • Machine<==>Language
    • Machine: Function/set of rules implied in hardwear. It is something that interprets language.
    • Language: Defines a machine.
  • Lexx: Unix program given adescription of a regular language, produces a C program (a finite state machine) that recognizes the language.
  • 4 Kinds of codes:
    1. Conveince: Unicode/ASCII-->Assigns numbers to symbols and text.
    2. Data compression: Zip, JPEG, MP3-->Strips redundancy to send more data.
    3. Error detection/correction system: Checksum-->Adds redundancy.
    4. Encryption
