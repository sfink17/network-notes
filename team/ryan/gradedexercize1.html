Graded Exercise 1

Questions

  1. What version of TCP does not use Fast Recovery as part of it's congestion handling?
    • TCP Tahoe. (p. 276)
  2. During the Slow Start of TCP congestion control the MMS amount starts at what value? How does it increment? When does it stop?
    • Slow start begins with an MMS transfer rate of 1 MMS. It adds 1 MMS each time a confirmation is received, this doubles the amonut each RTT. It stops when either a packet is dropped, or when triple duplicate ACK messages are received. If a packet is dropped it goes back to 1 MMS and sets the ssthresh to half of the cwnd when the drop occurred. Then it switches to congestion avoidance. If triple duplicate ACKs are received then it also goes back to 1 MMS and sets the ssthresh to half the cwnd when the triple ACKs arrived. However it switches over to fast recovery rather than congestion avoidance. (p. 272)
  3. What is the main difference between go-back-n, and selective repeat?
    • Go-back-n resends all packets after a missed packet, even ones that were received. Selective repeat allows the receiver to buffer out of order packets and will resend only those packets that aren't acknowledged. (p. 218-225)
  4. Advantages of UDP over TCP, disadvantages?
    • Advantages:
      • More flexibility
      • Faster speed
      • Light weight
    • Disadvantages:
      • Less reliable
      • No throttling
      • No constant connection
    • (p. 199-200)
  5. What is a 'bean' in regards to a java webserver?
    • A bean is a java class, one of the beans must take no parameters. (Lecture with Leon)
  6. Define DNS? What is it?
    • Domain name system. A data base that links hostnames to IP addresses. Distributed database and an API. Runs on Port 53 (sometimes port 80). (p. 130)
  7. Define load distribution.
    • For large websites with multiple IP addresses DNS rotates which address is used to avoid overloading one server. This balances the amount of users requesting each server. (p. 132)
  8. What is the amount of bytes in the UDP header? How many bytes per header field?
    • 8 bytes in the header total, and 2 bytes per header field. (Wireshark lab: UDP)
  9. Define queuing delay.
    • Delay that packet experiences when it waits to be transmitted onto the link/node. (p. 37)
  10. What two formats does the packet-contents window display the data of the captured packet in Wireshark?
    • ASCII and hexadecimal. (Wireshark lab: Intro)

Text Book Questions

  1. Suppose that the UDP receiver computer the internet checksum for the received UDP segment and finds that it matches the value carried in the checksum field. Can the receiver be absolutely certian that no bit errors have occurred? Explain. (p. 289, problem 5)
    • No. If the corresponding bits, that are normally added, for two of the 16-bit words are flipped when they are 0 and 1 or 1 and 0 then the sum is still the same. And the 1s complement would also be the same. The checksum would fail to realize that there is an error.
  2. Answer the following questions:
    (p. 293, problem 24)
    1. With the SR protocol, it is possible for the sender to receive an ACK for a packet that falls outside of its current window.
      • Yes, if the receiver's ACKs arrive after the sender times out then it will receive a second group ACKs for the resent packets, even though it would have advanced it's window past those packets.
    2. With GBN, it is possible for the sender to receive an ACK for a packet that falls outside of its current window.
      • Yes, this is more or less the same as it is in question 1. (difference being that it may get intermediate ACKs (ie. 2 and 3) in time, but still timeout for a previous one (ie. 1) and thus would resend 1 and get a second ACK after the first ACK had arrived and the window had incremented)
    3. The alternating-bit protocol is the same as the SR protocol with a sender and receiver window size of 1.
      • Yes.
    4. The alternating-bit protocol is the same as the GBN protocol with a sender and receiver window size of 1.
      • Yes, with a window size of 1 then there is no way that packets can occur out of order.
