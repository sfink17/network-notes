Lecture

  • Apache: Feather logo. "A patchy" program, name comes from a 'fix to a program', needed a lot of fixes. Webserver. Has a module to interpret php in a website. Port 80 (by default)
  • GlassFish: Webserver, displays java servers. Finds requested file, looks for 'special' java code, executes java, produces html, sends it back to client. Same way apache looks for php. May run with another webserver service. Port 88 (by default)
  • Use other softwear so when you see a request for a specific part of a url, redirect to other webserver (such as glassfish), and then get returned data to return to users.
  • Framework: Programming language, and a collection of classes that can be used. (ie. Ruby on rails, cake for php, jango for python).
  • Bean:Name of a java class in a java webserver network. One of the beans MUST take no parameters.
  • "Java webservers are difficult" -- Leon
  • Student Notes: UDP
    • Less reliable than TCP
    • More flexibility, speed, light weight
    • No throttling--all messages immediatly sent
    • No consistant connection
    • Used by steaming services,DNS, etc.
    • Uses checksum to detect errors
      • Adds 4 lines, 3 are 16 bit numbers, 4th is 1s compliment, checks to see if 1s compliment is accurate
      • possible for false positive, or false negative
  • Student Notes: 8.3.1 Reading notes
    • logical communication: From application perspective hosts are directly connected.
    • TCP: Reliable, connection-oriented service
    • UDP: Unreliable, connectionless service
    • IP: Best effort to deliver segments, no guarentee
    • Transport-layer multiplexing and demultiplexing: Extending host-to-host deliver to process-to-process. Both in TCP and UDP.
    • Congestion control: Prevents any 1 TCP connection from swapping links between communicating links
    • flow control: limits amount of data a sender can send. Keeps from over filling a reciever's buffer
    • Socket: dat passes from process to network
    • well known ports: 0-1023
    • Hand shake: before and fter data transmissions via TCP host sends a synchronized method to server, and the server sends back a synchronized method to open comunications
