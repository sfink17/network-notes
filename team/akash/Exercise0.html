  1. What is the role of a protocol in a network?

    A protocol defines the format and the order of messages exchanged between two or more communicating entities, as weel as the actions taken on the transmission and/or receipt of a message or other event. (Pg. 9)

  2. What is an Application Programming Interface in a network communications context?

    An API is a set of routines, protocols, and tools for building software applications. It specifies how a program running on one end system asks the Internet infrastrucutre to dilver data to a specific destination program running on another end system.(Pg. 6)

  3. What is Packet Sniffer?

    A packet sniffer is software that is used to capture, or intercept, the outgoing and incoming packets within a network for analyzation of the packet contents. (Pg. 58)

  4. Explain the difference between an client-server architecture and a P2P architeture

    A client-server architecture has an always on host (or a collection of them) which acts as a server. This server receives requests from various other hosts, which are called clients. The P2P architecture on the other hand involves direct communication between a pair of connected hosts called peers. Thus each host has the capacity to act as both a server or client, often both at the same time. (Pg. 86)

Donald Davis

The internet is best defined as a network of interconnected devices over a network, ranging from smartphones, laptops and computers to cars, locks and home appliances. However, the common factor over all these different devices is that the internet is driven by the excahnge of infromation, referred to as data, among these devices. This is called packet switching. However the concept of packet switching pre-dates the inception of the internet.

Packet switching was invented in the year 1965 by Doanld Watts Davies, a Welsh computer scientist. Born in 1924, he was employed at the UK National Physical Laboratory. Coincidentally, another computer scientist Paul Baran was developing a similar idea independently in the 1960s as well.

Davies was born in Rhondda Valley, Wales. He pursued his education at Imperial College London, and received a BSc degree in physics in 1943. After his studies he initially worked as an assistant to Klaus Fuchs, who ran the nuclear weapons project at Birmingham University. Soon though, Davies decided to return to Imperial College to gain a degree in mathematics. This lead to him being awarded the Lubbock Memorial Prize.

In 1966, while working at the National Physical Laboratory on the outskirts of London, Davies became interested in data communications. This was a result of a visit to MIT, where he noticed a significant problem. The computer systems were based on a new time-sharing architeture which involved keeping an open phone connection for each user. This was an extremely expensive proposition. His key observation was that computer network traffic was usually silent invovling long periods of silence, with certain "bursts" of data transfer. This eventually lead to the conception of packet switching which involved dividing computer messages into packets that are routed independently, and possibly via differing routes, across a network and are reassembled at the destination.

What is Git?

Git is basically a version control system (VCS). A Version control software is a system that records changes to a file or set of files over time so that you can recall specific versions later.

What is GitHub?

GitHub is a Git repository hosting service, but it adds many of its own features. While Git is a command line tool, GitHub provides a Web-based graphical interface. It also provides access control and several collaboration features, such as a wikis and basic task management tools for every project.

Basic Git Commands

Note, it is reccomended to learn a little bit about basic Linux terminal window commnds before trying to use git. This course is a good resource.

The following are the basic commandline inputs you can use to navigate through and use Git:

  • git init : creates a new git repository
  • git clone /path/to/repository : creates a working copy of a local repository by running the command
  • git add (filename) or git add * : can be used to propose changes
  • git commit -m "Commit message" : actually confirms and commits the changes
  • git push origin master : sends the changes to your remote repository. It is possible to select other branches by replacing the word master with the branch name.
  • git pull : updates your local repository to the newest commit
