Principles of Congestion Control

Packet loss typically results from the overflowing of router buffers as the network becomes congested. Packet retransmission is a way to treat the problem of network congestion, but does not solve the the underlying issue: many sources sending data at too high of a rate.

  • Available bit-rate(ABR) service
  • Asynchronous transfer mode(ATM) network

Vocab

  • Per-Connection througput: number of bytes per second at the reciever
  • Offered Load: The rate at which the transport layer sends segments into the network in bytes/second
  • End-to-end congestion control: no explicit support to transport layer.
  • Network-assisted congestion control: network-layer components provide explicit feedback to the sender regarding congestion state
  • Choke packet: direct feedback from network router to sender about congestion
  • Resource-management cells(RMC): used to convey congestion-related information among hosts and switches
  • Explicit forward congestion indication bit(EFCI bit): each data cell contains an EFCI bit. Set to one means there is congestion.
