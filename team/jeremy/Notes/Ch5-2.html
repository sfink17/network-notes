A network as a Link Layer

Multiprotocol Label Switching (MPLS): are fixed length headers that allowed for improved forwarding speeds from ISP in the mid-to-late 90's.

The MPLS Header inserted into a packet (above) is added when two devices capable of using the header communicate. The headers parts and uses are listed as follows:

  • A 20-bit label number. A label with the value of 1 represents the router alert label
  • A 3-bit Traffic Class field for quality of service priority and Congestion Notification, used to be called EXP bits
  • A 1-bit bottom of stack flag. If this is set, it signifies that the current label is the last in the stack
  • An 8-bit TTL field

This header sits between the Ethernet header and the IP header, or the layer 2 and layer 3 headers respectively. A router capable of MPLS is often called a label-switched router.
As far as this textbook is concerned the main and technical importance of MPLS's starts and ends with

  • The knowledge that MPLS's can be used to forward packets along routes that would otherwise be impossible to us
  • The ability to use MPLS's to avoid network congestion
  • The use of MPLS's in virtual private networks(VPNs)
