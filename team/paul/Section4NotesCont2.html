  1. Routing and Routing Algorithms

    • Least-Cost Path: A path among routers that is the most cost efficient, or takes the least cost.

    • Shortest Path: A shortest path may be a least-cost path. However, a shortest path algorithm tries to find the route with the shortest amount of links between the source and destination.

    • Global Routing Algorithms: These algorithms, called Link-State Algorithms, compute the shortest path between hosts using global knowledge about the network. This algorithm knows all states and costs of each link in the network.

    • Decentralized Routing Algorithm: These algorithms calculate the least-cost path without complete information, unlike the global routing algorithms. Rather, it finds the least-cost path through iterative processes.


  2. Characteristics of Routing Algorithms

    • Static Routing Algorithms: These routing algorithms change very slowly over time, usually because of some sort of human intervention.

    • Dynamic Routing Algorithms: These routing algorithms change dynamically based on traffic load, network congestion, or topology/cost changes.

    • There are also load-sensitive algorithms, which chooses to find routes around congested links that are still relatively short. If a router becomes majorly congested, then this characteristic of an algorithm will find a route around the congested link. Contrary to this, there are also load-insensitive algorithms which ignore congestion within links.
