Notes on Section 3

  1. What is Multiplexing?

    • Multiplexing is used in order to direct packets sent to and from hosts to specific processes on a host system (via ports).

    • Multiplexing is done by the host sending packets, in which the host specifies a source IP address (the sender), source port (for receiving), destination address, and destination port (to for demultiplexing to a specific process on the host system)

    • Multiplexing is useful, even when multiple hosts try to connect to a system, since multiplexing can still be done via the source IP addresses (they will be different for each host) even when the port is the same for all connections (for instance connecting to port 80).

  2. Multiplexing on UDP vs. TCP

    • In essence, TCP and UDP multiplexing work in the same way. However, there are key differences on how the multiplexing occurs based on the protocol.

      • ion
    • TCP uses reliable data transfer, congestion control, and requires a 3 way handshake (to establish a reliable connection

    • UDP doesn't have congestion control, reliable data transfer, or even have a way to show that a connection was established. UDP has a connectionless (that is, no handshake) transport scheme.

  3. Why use UDP then?

    • Although UDP is seemingly less effective, it does use less data to use multiplexing. TCP uses 20 bytes of overhead data, whereas UDP uses only 8 bytes.
