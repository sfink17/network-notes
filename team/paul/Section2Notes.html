Notes on HTTP (Section 2)

  1. HTTP is called the Hypertext Transfer Protocol
    • Used in practice to get website files (.html files, .shtml files, etc.) which are then displayed in the browser
    • Client performs a request in HTTP:

      GET /index.html HTTP/1.1
      Host: www.google.com

    • Server Responds with an acknowledgement:

      HTTP/1.1 200 OK
      Date: Mon, 27 Jul 2009 12:28:53 GMT
      Server: Apache/2.2.14 (Win32)
      Last-Modified: Wed, 22 Jul 2009 19:15:56 GMT
      Content-Length: 88
      Content-Type: text/html
      Connection: Closed


    Types of Requests
    • GET: Grabbing a specific file from the webpage (which is most likely a .html, .shtml, etc.) for the browser to display

    • POST: Sending information to grab a file from the webpage, but after information is submitted (the client POSTS something to webpage first)

    • PUT: Client is putting a file on a webserver

    • DELETE: Client is requesting a deletion of a file on a webserver


  2. User-Agent Usefullness:
    • User-Agent: When client requests a webpage via a browser, a user-agent is sent to tell the webserver how to display the data that it transmits

    • Ex. Mozilla 5/0, Gecko/30909, etc.


  3. Cookies and Us
    • Cookies are often sent to the user in a response request (or after a POST request) to help the browser remember who the user was when the website was visited

    • Example of cookie getting set:
      HTTP/1.1 200 OK
      Date: Mon, 27 Jul 2009 12:28:53 GMT
      Server: Apache/2.2.14 (Win32)
      Last-Modified: Wed, 22 Jul 2009 19:15:56 GMT
      Content-Length: 88
      Content-Type: text/html
      Connection: Closed
      Set-Cookie: 345234

  4. HTTP vs. HTTPS
    • HTTP often sends data in cleartext, so those sniffing any HTTP request or response packets will be able to clearly see the data being transferred (even if it's sensitive)

    • HTTPS encrypts the transport layer of a packet, making it hard to see the cleartext of the website responses because it because it becomes encrypted in SSL

  5. HTTP is stateless
    • HTTP is terminated after the client is done interacting with the webserver (either non-persistent or persistent), unless cookies are used or placed in the browsers cache.
