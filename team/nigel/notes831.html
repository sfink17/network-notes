Notes 8.3.1


Logical Communication - From an application's perspective, hosts are directly connected.
Application message -> segment
TCP (Transmission Control Protocol) - reliable, connection-oriented service
UDP (User Datagram Protocol) - unreliable, connectionless service
IP (Internet Protocol) is a best-efford delivery service - It makes its "best effort" to deliver segments, but offers no guarantee.
IP is said to be an unreliable service.
transport-layer multiplexing and demultiplexing - Extending host-to-host delivery to process-to-process delivery.
TCP offers reliable data transfer and congestion control.
Congestion Control - prevents any one connection from swamping the links and routers between hosts.
A process may have more than one socket.
socket - where data passes from process to network
well-known port numbers - ports 0-1023.

UDP

  • Does as little as possible.
  • No Handshaking - connectionless
  • Useful for applications that are okay with data loss
  • Small packet overhead
  • There is a checksum
end-end principle - error detection on end-to-end basis.
